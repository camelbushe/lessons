# Хэш таблицы
Структура данных, которая хранит элементы в виде пар ключ-значение.
## Что такое хэш функция
**Хэш функция** - это функция, которая принимает строку и возвращает число, которое будет соответствовать этой строке.
### Требования
- Для одной и той же строки всегда возвращает одно и тоже число.
- Одно и тоже число не может соответствовать ещё каким либо строкам.
## Суть
Хэш таблиц не существует в компьютерах. Это что-то абстрактное, но вот как она реализуется.
#### При добавлении
1. Принимаем строку
2. Пропускаем через хэш функцию
3. Сохраняем в массиве значение для соответствующей строки под индексом, который вернула функция.
#### При возврате
1. Принимаем строку
2. Пропускаем через хэш функцию
3. Обращаемся к массиву по индексу который вернула функция и получаем значение для переданной строки.

# Применение хэш таблиц
- Кэширования. Храним под ключами значения и если они существуют в таблице, то получаем из нее
```python
cache = {}

def getPage(url):
    if url in cache:
        return cache[url]
    else:
        page = generatePage(url)
        cache[url] = page
        return page
```
Сдесь мы получаем страницу по url. Если мы уже переходили по этому url, то страница уже сгенерирована и нам не нужно ее генерировать (что занимает много времени).
- Устранение дубликатов
```python
voted = {}

def check_vote(name):
    if name in votes:
        print("Не допускать")
    else:
        voted[name] = True
        print("Допустить")
```
Допустим мы проводим голосование. Голосовать нужно 1 раз. Если голос уже есть, то повторно голосовать не нужно
- Моделирование отношений между объектами
```python
phone_book = {}

phone_book["Bob"] = "235467"
phone_book["Mark"] = "729639"
```
Храним номера по имени пользователя


# Коллизии в хэш таблицах
Реализовать такую хэш функцию, которая будет выдавать уникальное число на каждый элемент **при условии, что размер массива не бесконечный** (т.е. память ограничена, а комбинаций ключей очень очень много) пратичеси не возможно. Поэтому могут возникать коллизии. Рассмотрим пример. У нас есть хэш таблица, а данные в массив указываются **в зависимости от первой буквы ключа**.
```text
[Тут буква а][Тут буква б][][][][]...[Тут буква я]
```
Если ключ апельсин, то значение для него помещается в первую ячейкку. Для яблока в последнюю. Но что если мы ещё добавим авокадо. Вознинет колизия. Чтобы это решить, в ячейку с оллизией помещают связанный список.
```text
[Коллизия] -> [Апельсин][0.63] -> [Авакадо][0.56]
[0.78 (цена банана)]
[]
...
```
Каждый узел связанного списка на месте коллизии хранит люч, значение и ссылку на следущий элемент.
## Коэффициент заполнения
Вычисляется по следущей формуле
```math
Load Factor = \frac{m}{n}
```
Где:
* $m$ - количество элементов в хэш таблице
* $n$ - общее количество элементов
```text
[][1][][0][]

LoadFactor = 0.4
```
По мере роста хэш таблицы приходится изменять ее размер. Это желательно делать когда кофициент заполнения достигает 0.7. Обычно в этом случае создаётся массив в двое больше предыдущего и в него переносятся элементы при помощи хэш функции.
```text
[4][1][3][] - Приблежается к заполнению
[][][][][][][][] - Создаём массив в двое больше
[][4][][1][3][][][] - Переносим элементы при помощи хэш функции
```
